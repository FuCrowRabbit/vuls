fast scan

fast scanは、スキャン対象サーバにおけるroot権限や依存を必要とせずにスキャン出来ます。 詳しくは、以下を参照してください。 実際にスキャンする前に、vuls configtestで設定をチェックしてください。 詳しくは、以下を参照してください。

  • Architecture/fast
  • Configtest/fast scan

インターネット接続を使うfast scan

  • config.toml
[servers]

[servers.localhost]
host         = "192.168.100.111" # or "127.0.0.1"
port         = "22"
scanMode     = ["fast"]

インターネット接続を使わないfast scan

  • config.toml
[servers]

[servers.localhost]
host         = "192.168.100.111" # or "127.0.0.1"
port         = "22"
scanMode     = ["fast", "offline"]

deep scan

  • 今のところfast-rootスキャンモードと同じです。

-ssh-native-insecure オプション

Vulsは複数のSSH接続方法をサポートしています。

デフォルトでは、外部SSHコマンドを使います。 Goによる実装ではサポートしていないChiperアルゴリズムやProxyCommandを使いたい場合は、外部SSHコマンドを利用すると便利です。
スキャン対象サーバの/etc/sudoersに以下の行を追加しておくことを忘れないようにしてください。 (username: vuls)

Defaults:vuls !requiretty

GoライブラリのSSH実装であるcrypto/sshを使いたい場合はこのオプションを指定してください。
既存のUNIXツールを使えない状況で便利です。 ただし、このモードではhost keyをチェックしないことに注意してください。

-ask-key-password オプション

SSH key password-ask-key-password
empty password-
with passwordrequiredor use ssh-agent

例: 設定ファイルに定義された全サーバをスキャンする

$ vuls scan -ask-key-password

このサンプルコマンドでは、

  • スキャン前にSSH秘密鍵のパスワードを入力
  • 設定ファイルに定義された全サーバをスキャンする

例: 設定フィアルに定義されたサーバのうち特定のサーバをスキャンする

$ vuls scan server1 server2

このコマンドが意味することは...

  • SSH鍵認証、パスワードが設定されてない秘密鍵を用いる（-ask-key-passwordオプションを指定していないので）
  • 2台のみスキャンする（server1, server2)

例: SSH経由ではなく、コマンドを直接実行してスキャンする

設定ファイルのhostに、localhst or 127.0.0.1が指定されている、かつ、portにlocalが指定されているサーバは、SSH経由ではなく直接コマンドを発行してスキャンする。詳細は、Architecture section参照。

  • config.toml

    [servers]

[servers.localhost]
host         = "localhost" # or "127.0.0.1"
port         = "local"

例: 実行中のコンテナをスキャンする (Docker/LXD/LXC)

コンテナ内でsshdデーモンを起動しないことが普通です。 Docker Blog:Why you don't need to run SSHd in your Docker containersも参照してください。

Docker

Vulsは実行中の Docker コンテナを、SSHではなく docker exec を用いてスキャンします。 詳しくは Architecture sectionを参照してください。

もしrootを使いたくない時は、Unixグループを作ってユーザをそこに追加してください。詳細は、docker manualを参照してください。

設定されたすべての実行中のコンテナをスキャンする

設定ファイルに"${running}"が設定されている必要があります。


[servers.172-31-4-82] host = "172.31.4.82" user = "ec2-user" keyPath = "/home/username/.ssh/id_rsa" containerType = "docker" containersIncluded = ["${running}"] ```

#### 特定の実行中のコンテナだけをスキャンする

コンテナIDかコンテナ名が設定ファイルに設定されている必要があります。  
以下の例では、`container_name_a` と`4aa37a8b63b9`のみがスキャンされます。   
スキャン前に、これらのコンテナが実行中であることを確認してください。  
もし指定されたコンテナが実行中ではない場合、Vulsはエラーメッセージを出してスキャンを終了します。

``` [servers]

[servers.172-31-4-82] host = "172.31.4.82" user = "ec2-user" keyPath = "/home/username/.ssh/id_rsa" containerType = "docker" containersIncluded = ["container_name_a", "4aa37a8b63b9"] ```

#### 特定の実行中のコンテナ以外だけをスキャンする

```toml
[servers]

[servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"
containerType = "docker"
containersIncluded = ["${running}"]
containersExcluded = ["container_name_a", "4aa37a8b63b9"]

コンテナだけをスキャンする (Dockerホストはスキャンされない)

--containers-onlyオプションが使えます。

LXD

LXDをスキャンするときは、SSHではなく lxc execを使用してコマンドを実行してスキャンします。

[servers]

[servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"
containertype = "lxd"
containersIncluded = ["${running}"]
containersExcluded = ["container_name_a", "4aa37a8b63b9"]

LXC

lxcをスキャンするときは、SSHではなく lxc-attachを使用してコマンドを実行してスキャンします。

[servers]

[servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"
containertype = "lxc"
containersIncluded = ["${running}"]
containersExcluded = ["container_name_a", "4aa37a8b63b9"]

LXCのスキャンはroot権限が必要です。

スキャン対象サーバ上の/etc/sudoersの例

vuls ALL=(ALL) NOPASSWD:SETENV: /usr/bin/lxc-attach -n *, /usr/bin/lxc-ls *

例: WordPressのスキャン (本体、プラグイン、テーマ)

詳しくは、usage-scan-wordpressを参照してください。
